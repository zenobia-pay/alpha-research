**Category Notes**
- **Product confusion:** Moderate. The CLI says it inspected the dataset, but it never tells the user whether the requested analysis is actually underway or how far along it is.
- **Dataset confusion:** Low. The dataset name `econ` is clear, but the CLI does not confirm any field resolution or inferred schema details.
- **Auth confusion:** None visible in this journey.
- **Run lifecycle confusion:** High. The system switches from inspection to a busy-run block without clarifying queueing, ownership, or expected retry behavior.
- **Terminal/UI readability:** Moderate. The long dashboard URL and run id wrap awkwardly, and the repeated status lines consume most of the screen without adding new information.

**Missing Information That Would Have Helped**
- Whether the active run belongs to the current user/session.
- Whether the CLI will retry automatically or if the user must rerun later.
- Any ETA or expected wait condition for the busy dataset.
- A short confirmation of what the inspection found, especially if field names were being resolved.
- A compact summary of expected artifacts, such as table, scatter plot, and markdown summary, once the run can start.

**Information That Should Be Removed or De-emphasized**
- Repeated unchanged progress lines after the first few seconds.
- The full dashboard URL in-line, or at least its current visual prominence.
- The raw `research debug run ...` suggestion in the main flow, unless the user explicitly asked for debugging.
- `Found 7 remote datasets.` if it does not affect the current decision.

**Suggested UI/Output Changes**
- Show a single, compact status card for async work: selected dataset, current phase, blocker, and next action.
- When a dataset is busy, say explicitly whether the user should wait, retry, or inspect the active run.
- Replace the raw debug command with a more human next step, and optionally tuck the debug command behind a secondary hint.
- Compress long operational details into a secondary panel or a keybinding-driven details view.
- If inspection is only preflight, say so plainly, for example: “Resolved dataset `econ`; checking availability before starting analysis.”
