- **Auth confusion:** None observed. There was no login prompt, error, or permission failure.
- **Run lifecycle confusion:** The output mixes active booting runs and a completed run without clearly separating “current work in progress” from “the run you asked for.”
- **Terminal/UI readability:** The text is legible, but the repeated status lines, long URLs, and wrapped UUIDs make the output feel heavier than it needs to be.

**Missing Information That Would Have Helped**

- A one-line explanation of why the selected run was chosen.
- A compact chooser if there are multiple plausible candidates.
- A direct preview of the completed run’s result, not just artifact labels.
- A timestamp or relative age for the selected run, so “last run” is anchored in time.
- Clearer status labeling for active vs completed runs.

**Information That Should Be Removed or De-Emphasized**

- Full dashboard URLs in the main conversational output.
- Raw run UUIDs unless the user explicitly asks for them.
- Repeated unchanged progress text after the first status update.
- The generic “Artifacts are the saved outputs from the run” line, which adds little once artifact names are already shown.

**Suggested UI/Output Changes**

- Show the chosen run first, with a reason line: “Selected the most recent completed run.”
- If there are multiple candidates, show a small numbered choice list instead of a mixed status dump.
- Replace raw URLs with compact action labels like “Open dashboard” and “Open transcript.”
- Surface a short preview of the summary artifact inline so the user sees actual results immediately.
- Keep a visible spinner or heartbeat during the 15-second wait so the CLI feels alive.
- Collapse unchanged lines after the first refresh to reduce noise.
