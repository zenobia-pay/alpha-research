**Information To Remove Or De-Emphasize**
- The environment provisioning promise at the end. It makes the assistant sound like it is already taking action before the user answers.
- The full four-item dataset list could be compressed into a primary recommendation plus 1-2 optional supplements.
- “All available in RESEARCH” is not wrong, but it adds little value for the user at this stage.

**Suggested UI/Output Changes**
- Show an immediate “looking up candidate datasets” or “analyzing topic” status so the screen is not blank for 15 seconds.
- Present a structured format:
  - `Primary dataset`
  - `Why`
  - `Optional supplements`
  - `Need from you`
- If multiple datasets are plausible, ask one focused question first instead of bundling every clarifier.
- Do not mention environment reuse/creation until after the user confirms the dataset direction.
- If the CLI is actually retrieving metadata, surface that explicitly so the user understands the delay is work, not a hang.
