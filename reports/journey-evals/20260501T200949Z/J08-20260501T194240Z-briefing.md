**What Was Missing**
- A shorter, more explicit set of choices for virality definition.
- A clearer decision point for the user, such as:
  - engagement percentile vs absolute threshold,
  - sample size,
  - whether the analysis should prioritize text, media, or topic breakdowns.
- A note that the dataset choice is a proposal, not an assumption, if `enriched-tweets` is not guaranteed.

**What Should Be Removed or De-emphasized**
- De-emphasize the long list of planned labels in the first message.
- De-emphasize the speculative output details (`visualizations`, `representative examples`) until after the user confirms the experiment frame.
- Trim the `if available` phrasing or move it into a shorter parenthetical so it does not distract from the main question.

**Suggested UI/Output Changes**
- Compress the clarification into one compact proposal plus one explicit question.
- Present 2-3 concrete virality definitions instead of only one suggested threshold.
- Separate “experiment design” from “confirmation” more clearly:
  - brief summary,
  - proposed metric,
  - proposed sample size,
  - one yes/no or choose-one question.
- Improve wrapping so that long phrases do not split awkwardly in the terminal viewport.
