- Dashboard and terminal URLs as the primary visible result.

**Suggested UI / Output Changes**
- Show a one-paragraph proposed plan before any remote run starts.
- Ask one explicit confirmation question when the prompt is vague, such as “Do you want me to define virality as X, sample N tweets, and compare Y labels?”

- Replace raw tool chatter with user-facing states like `scoping`, `drafting plan`, `waiting for approval`, `running`, and `done`.
