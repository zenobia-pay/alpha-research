Alpha Research currently publishes one canonical public dataset: `econ`.

Canonical datasets are remote mounted raw public source packages. The dataset owns a comprehensive `dataset_briefing.md` that the CLI can read through `dataset.briefing.markdown`. The docs page mirrors that briefing file so the public documentation carries the same current understanding of exact raw source files/API responses, native source shapes, coverage, provenance, quality checks, and gaps.

| Dataset | ID | Current raw-source profile | Source families | Canonical shape |
| --- | --- | --- | ---: | --- |
| [Econ](/public-datasets/econ) | `econ` | Economics, housing, labor, credit, and macro-finance | 40 | raw provider files/API responses |

## Refresh contract

Every canonical refresh should publish:

- `manifest.json`
- `source_registry.csv`
- `source_registry.plan.json`
- `download_inventory.jsonl` and `download_inventory.csv`
- `raw_inventory.jsonl` and `raw_inventory.csv`
- `data_dictionary.md`
- `quality_report.md`
- `dataset_briefing.md`
- `docs/public-datasets/briefings/<datasetId>.md`
- `docs/public-datasets/<datasetId>.mdx`

Canonical datasets should not publish analysis-ready tables, merged panels, shared entity models, cross-source joins, or derived fields. They should preserve the exact native shape of each source-specific file/API response/document collection, with provenance and source-shape details in the briefing.
