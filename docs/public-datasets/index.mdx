Canonical datasets are remote mounted research environments. Each dataset owns a comprehensive `dataset_briefing.md` that the CLI can read directly. The docs pages mirror those briefing files so the public documentation carries the same current understanding of exact source files, table/document shapes, schemas, coverage, transformations, quality checks, and gaps.

## Refresh contract

Every canonical refresh should publish:

- `manifest.json`
- `source_registry.csv`
- `source_registry.plan.json`
- `download_inventory.jsonl` and `download_inventory.csv`
- `normalization_inventory.jsonl` and `normalization_inventory.csv`
- `data_dictionary.md`
- `quality_report.md`
- `dataset_briefing.md`
- `docs/public-datasets/briefings/<datasetId>.md`
- `docs/public-datasets/<datasetId>.mdx`

Canonical datasets should not be forced into one universal normalized table. They should preserve the exact shape of each source-specific file, table, document collection, and derived panel, with provenance and transformation details in the briefing.
