---
title: "Runs in the Dashboard"
description: "Track remote research runs and inspect their artifacts."
---

A run is a remote job against a dataset. Runs can query, aggregate, transform, label, analyze, document, or debug a research environment.

The runs view helps you:

- distinguish completed, active, blocked, failed, and cancelled work
- open run events and status
- inspect artifacts produced by remote agents
- follow dashboard run links from CLI output

Canonical dashboard run links use this route shape:

```text
https://dashboard.alpharesearch.nyc/?view=runs&runId=<run-id>#run-<run-id>
```
