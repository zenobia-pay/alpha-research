Alpha Research supports four main workflows. They map to the main jobs of `research`: intake messy data, navigate and trust datasets, design concrete studies, and operate remote research runs through completion.

<CardGroup cols={2}>
  <Card title="Use public datasets" icon="database" href="/ways-to-use/public-datasets">
    Start from canonical datasets like Econ, Sociology, Philosophy, and Classics.
  </Card>
  <Card title="Bring private data" icon="lock-keyhole" href="/ways-to-use/private-datasets">
    Upload a private CSV or document corpus and create a workspace-scoped research environment.
  </Card>
  <Card title="Run research jobs" icon="zap" href="/ways-to-use/research-runs">
    Launch remote analysis, transformation, labeling, and documentation runs.
  </Card>
  <Card title="Develop locally" icon="code" href="/ways-to-use/local-development">
    Run the API, frontend, ingest tooling, and fixture datasets from this repository.
  </Card>
</CardGroup>
