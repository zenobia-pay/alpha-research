Transformations

value
Each raw API response stored under dataset/raw/*.json was flattened and mapped into a 17-column normalized schema.
Source categories and record_type values were harmonized into controlled vocabularies (e.g., newspaper_issue, gazetteer_reference).
A single Parquet table aggregates all sources with consistent column ordering and types; CSV and JSON derivatives provide previews and source registry metadata.
Supplemental attributes that did not fit canonical fields were serialized into the extra_notes JSON string.

Limitations

value
Harvested samples are truncated (e.g., first five LOC pages, 200 Wikidata events) and do not represent complete source catalogs.
temporal_coverage mixes ISO timestamps, textual dates, and empty values; consumers must normalize per source.
record_id duplication is common for Library of Congress collections and issues due to multiple entries referencing the same URL.
Licensing statements are copied verbatim from sources and may require manual verification before redistribution.
extra_notes retains JSON-encoded strings that require parsing for structured use.
